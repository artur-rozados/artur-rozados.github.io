Description

The Goal

The primary goal of this project was to create a realistic and manageable platform for conducting Federated Learning experiments. Research in this area requires testing algorithms on a variety of hardware-constrained devices that mimic real-world 'edge' environments‚Äîa task that is notoriously difficult to scale and reproduce with manual configurations.

My Solution

I designed and built a heterogeneous cluster of 18 mini-computers (including Raspberry Pis and NVIDIA Jetsons) operating on an isolated subnet. To manage this complex environment, I developed a comprehensive set of Ansible playbooks that automates the entire lifecycle of the testbed‚Äîfrom initial OS provisioning and software installation to network configuration and experiment deployment.

This "Infrastructure as Code" approach transforms a complex, error-prone manual process into a reliable, push-button solution.

Key Features & Impact

  • Automated: Eliminates repetitive manual setup, allowing researchers to focus on experiments instead of infrastructure.
  • Scalable & Reproducible: Ansible ensures that every experiment is perfectly reproducible and the entire environment can be easily scaled or rebuilt.
  • Heterogeneous Environment: Simulates real-world conditions by allowing algorithm performance testing across diverse hardware with varying constraints (CPU, power consumption, thermals).
  • Accelerated Research: Drastically reduces the time required to set up and run new experiments, speeding up the overall research and development cycle.
